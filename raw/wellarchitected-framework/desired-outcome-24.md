# Desired outcome:

• You create temporary duplicates of environments, which lowers the risk, effort, and cost of
experimentation and testing.
• These duplicated environments can be used to test the conclusions from your analysis,
experiment, and develop and test planned improvements.
• You run gamedays, and you use Fault Injection Service (FIS) to provide the controls and
guardrails that teams need to run experiments in a production-like environment.
