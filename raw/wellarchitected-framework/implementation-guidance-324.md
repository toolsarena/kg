# Implementation guidance

Build and deployment management systems are used to track and implement change, reduce
errors caused by manual processes, and reduce the level of effort required for safe deployments.
Fully automate the integration and deployment pipeline from code check-in through build,
testing, deployment, and validation. This reduces lead time, decreases cost, encourages increased
frequency of change, reduces the level of effort, and increases collaboration.
