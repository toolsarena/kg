# Related workshops:

• AWS Transit Gateway Workshop
REL02-BP05 Enforce non-overlapping private IP address ranges in all private address spaces
where they are connected
The IP address ranges of each of your VPCs must not overlap when peered, connected via Transit
Gateway, or connected over VPN. Avoid IP address conflicts between a VPC and on-premises
environments or with other cloud providers that you use. You must also have a way to allocate
private IP address ranges when needed. An IP address management (IPAM) system can help with
automating this.
