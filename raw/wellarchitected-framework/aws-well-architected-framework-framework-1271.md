# AWS Well-Architected Framework Framework

A good hypothesis is based on how the workload is expected to mitigate the fault to maintain
the steady state. The hypothesis states that given the fault of a specific type, the system
or workload will continue steady state, because the workload was designed with specific
mitigations. The specific type of fault and mitigations should be specified in the hypothesis.
