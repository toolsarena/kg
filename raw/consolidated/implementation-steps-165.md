# Implementation steps

• Estimate the bandwidth and latency requirements of your existing applications.
• For existing workloads that are moving to AWS, leverage the data from your internal network
monitoring systems.
• For new or existing workloads for which you don’t have monitoring data, consult with
the product owners to determine adequate performance metrics and provide a good user
experience.
• Select dedicated connection or VPN as your connectivity option. Based on all workload
requirements (encryption, bandwidth, and traffic needs), you can either choose AWS Direct
Connect or Site-to-Site VPN (or both). The following diagram can help you choose the
appropriate connection type.
