# Common anti-patterns:

• Your production workload exists only in a single Availability Zone.
• You implement a multi-Region architecture when a multi-AZ architecture would satisfy business
requirements.
• Your deployments or data become desynchronized, which results in configuration drift or under-
replicated data.
• You don't account for dependencies between application components if resilience and multi-
location requirements differ between those components.
