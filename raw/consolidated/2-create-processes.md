# 2. Create processes:

• Develop a process to assess which events are significant and require monitoring. This involves
setting thresholds and parameters for normal and abnormal activities.
• Determine criteria escalating an event to an incident. This could be based on the severity,
impact on users, or deviation from expected behavior.
• Regularly review the event monitoring and response processes. This includes analyzing past
incidents, adjusting thresholds, and refining alerting mechanisms.
